### 2. Material Symbols 图标验证

**重要**：Material Symbols 图标必须显示为图形符号，而不是文本字符串。

检查以下位置的图标是否正确显示：
- ✅ 左侧面板：`auto_awesome`（应显示为星形/闪光图标）
- ✅ 用户名输入框：`person`（应显示为人形图标）
- ✅ 密码输入框：`visibility`（应显示为眼睛图标）
- ✅ 校园统一认证按钮：`school`（应显示为学校/建筑图标）
- ✅ 微信扫码按钮：`qr_code_scanner`（应显示为二维码扫描图标）

**注意**：如果图标显示为文本（如 "school", "person" 等），说明 Material Symbols 字体未正确加载。请检查：
1. `app/layout.tsx` 中是否包含 Google Fonts 的 preconnect 和 stylesheet 链接
2. `app/globals.css` 中是否包含 `.material-symbols-outlined` 的 font-variation-settings 样式
3. 浏览器是否阻止了外部字体加载（检查浏览器控制台）

### 3. 功能验证

#### 登录流程测试
1. 输入有效的学号/用户名和密码
2. 点击"立即登录"按钮
3. 验证按钮显示"登录中..."状态
4. 成功后验证跳转：
   - 学生角色 → `/student/course`
   - 教师角色 → `/teacher/behavior`

#### 响应式测试
- 移动端：左侧蓝色区域应隐藏
- 桌面端：左右分栏显示

## 实现说明

### 保持的差异（仅限必要的 React 适配）
1. `<form>` 添加了 `onSubmit` 事件处理器
2. 输入框添加了 `value` 和 `onChange` 绑定（React 受控组件）
3. 按钮改为 `type="submit"` 并添加了 `disabled` 和加载状态
4. 错误提示区域（原型中没有，但登录功能需要）

### 保持完全一致的部分
- 所有 Tailwind class 名称和顺序
- 所有文本内容（中文文案）
- 所有图标名称
- DOM 元素层级结构
- 背景渐变和阴影效果
- 间距、字体大小、颜色

## 问题排查

### Material Symbols 图标不显示
**症状**：图标显示为文本字符串（如 "school", "person"）

**原因**：
- Google Fonts 未正确加载
- 浏览器内容阻止器阻止了外部资源
- 网络连接问题

**解决方案**：
1. 检查浏览器开发者工具的 Network 标签，确认字体文件是否加载成功
2. 临时禁用内容阻止器/广告拦截器
3. 检查 `app/layout.tsx` 中的字体链接是否正确
4. 清除浏览器缓存后重新加载

### 样式差异
如果发现样式与原型不一致：
1. 使用浏览器开发者工具检查元素的实际 class
2. 对比原型 HTML 和实现的 TSX 文件
3. 确认 Tailwind 配置正确（tailwind.config.ts 和 globals.css）

## 验收标准

- ✅ Material Symbols 图标正确显示为图形符号
- ✅ 登录功能正常工作（提交表单，角色跳转）
